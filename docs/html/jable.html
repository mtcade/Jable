The JyFrame is a simple, unassuming representation of the Json Table (jable) has rows as numbers, columns as strings. There are two reasons to use it over .csv:

  1. It can encode anything serializable to Json as an item in a cell, with support for custom encoders and decoders

  2. It can save space by not repeating the same values over and over

Otherwise, it does not offer many powers other tabular data do not.

class jable.JyFrame(fixed: dict[str, any] = {}, shift: dict[str, list] = {}, shiftIndex: dict[str, list] = {}, keyTypes: dict[str, type] = {}, meta: dict[str, any] = {}, customTypes: dict[str, type] = {})¶

Stores column data as a combination of three parts:

Parameters:

  • fixed (dict[ str, any ]) – Columns with the same name for value in every row. You can change its value but that will essentially change it for every row; try to only change it in code that knows it’s a fixed key, and sets and gets it with the fixed methods (.keys_fixed(), .get_fixed(…))

  • shift (dict[ str, list ]) – Where we have a listed literal item as a value, typically a string, float or int, sometimes lists of them, sometimes dictionaries or any other objects

  • shiftIndex (dict[ str list ]) – Sometimes, shift columns will have the same value repeated many times. If its column name is in shift index, then the values in shift are integers, referring to the index in shiftIndex of the same column. I.e., if we have shift[“fur_color”] = [1,0,2] and shiftIndex[“fur_color”] = [“green”,”orange”,”purple”,”red”], then the “fur_color”`s are really `[“orange”,”green”,”purple”]

  • keyTypes (dict[ str, str | type ]) – Optional types to set for any columns. No enforcement is done by the JyFrame itself, whether inserting or retriving. It is for your own use. These will be serialized as strings using str so add the appropriate functions for custom classes to serialize it as you want, and convert into a class upon reading. Includes support for basic python types

  • meta (dict[ str, any ]) – Another arbitrary dictionary to hold domain specific data, in the jyFrame as ._meta. No methods write or use this, so edit and read at will or subclass.

  • customTypes (dict[ str, type ]) – A reference to use for deserializing string types from keyTypes. Gets checked before builtin types

For jyFrame: JyFrame You can get a column my_column: str of data by taking jyFrame[my_column]. You can get a row j: int as a dictionary with jyFrame[j]. You can get one item with jyFrame[j,my_column]; see .__getitem__(…)

append(row: dict[str, any], strict: bool = True) → None¶

Add a dictionary as a row dict as the last index

If strict, append row which has keys a subset of the keys of self._shift If not strict, append keys from row which are present in self._shift, and None for keys missing from row
